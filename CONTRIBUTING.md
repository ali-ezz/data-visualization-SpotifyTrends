# Contributing

Thank you for contributing to this repository. Please follow these guidelines when opening issues or pull requests.

## How to contribute

1. Fork the repository.
2. Create a new branch with a descriptive name.
3. Add a clear summary and any relevant tests.
4. Open a pull request with a concise description of your changes.

## Branch naming

Use descriptive branch names such as:
- `feat/add-dashboard-filter`
- `fix/data-cleaning-error`
- `docs/update-readme`

## Testing

Run the Python unit tests with:

```bash
pytest
```

## Code style

- Keep changes small and focused.
- Add tests for new behavior when possible.
- Keep documentation updated for installation and usage.
